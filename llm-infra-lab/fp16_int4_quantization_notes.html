FP16 vs INT4 量化推理实验笔记

围绕显存占用、加载速度、推理速度和回答质量，对比普通 FP16 推理与 bitsandbytes INT4 量化推理。本文把实验结果、技术原理和面试表达放在一起，方便复盘。

1. 一句话结论

INT4 量化的核心收益是降低权重存储精度，从而显著减少显存占用；代价是加载和推理时可能引入量化层构建、反量化、低比特 kernel 等额外开销。因此 INT4 通常更省显存，但不保证一定比 FP16 更快，回答质量也需要结合任务人工评估。

显存占用 -67% 3.19GB 到 1.04GB
加载速度 INT4 更慢 2.85s 到 4.52s
推理速度 -21% 53.61 到 42.32 tokens/s
回答质量 无崩坏 但准确性受模型能力限制

2. FP16 与 INT4 的本质差异

FP16 推理

  • 权重通常以 16-bit 浮点数存储和参与计算。
  • GPU 对 FP16 Tensor Core 支持成熟，推理速度通常稳定。
  • 缺点是权重显存较高，模型越大越容易受显存限制。

INT4 量化推理

  • 权重以 4-bit 形式压缩存储，常见方案包括 NF4。
  • 显存占用明显下降，更适合消费级 GPU 或多模型部署。
  • 计算时可能需要反量化到 FP16/BF16，速度不一定提升。
本实验使用 Transformers + bitsandbytes 的 4bit 加载方式：load_in_4bit=True、bnb_4bit_quant_type="nf4"、bnb_4bit_compute_dtype=torch.float16。也就是说，权重用 4bit 存储，但计算 dtype 仍以 FP16 为主。

3. INT4 推理链路

INT4 不是简单地把所有计算都变成 4bit。更常见的流程是：权重以低比特形式存放在显存中，矩阵乘法前由量化 kernel 读取 4bit 权重，并结合 scale/codebook 恢复近似值，再与激活做计算。这样可以减少权重读取量，但会增加反量化和 kernel 调度成本。

5. 实测结果

6. 显存占用为什么下降

大模型推理时，显存并不只包含权重，还包含激活、KV Cache、CUDA allocator reserved memory、临时 buffer 等。INT4 主要压缩的是权重部分，所以当模型权重是主要显存来源时，收益非常明显；当上下文很长、batch 很大时，KV Cache 和激活也会成为重要显存来源。

7. 加载速度与推理速度怎么理解

加载速度

FP16 加载路径较直接：读取权重、放到设备、构建模型模块。INT4 加载除了读取权重，还需要替换或构建量化线性层，处理量化参数，并完成 device map 分配。因此第一次排除下载影响后，仍可能出现 INT4 加载慢于 FP16 的现象。本实验中 FP16 为 2.85s，INT4 为 4.52s。

推理速度

推理速度取决于计算瓶颈和访存瓶颈的平衡。INT4 降低了权重读取量，理论上对访存瓶颈有帮助；但如果模型较小、batch 较小、GPU 的 FP16 路径已经很快，那么反量化和低比特 kernel 开销可能抵消收益。本实验中 INT4 从 53.61 tokens/s 下降到 42.32 tokens/s，说明当前配置下 INT4 更像是“省显存方案”，不是“加速方案”。

面试里不要绝对说“INT4 一定更快”。更准确的说法是：INT4 通常降低显存和访存压力，但速度取决于硬件、kernel 实现、模型大小、batch size、上下文长度和推理框架。

8. 回答质量分析

两种模式都能输出完整英文回答，没有明显乱码、格式崩坏或重复循环；但它们对 KV Cache 的解释都不够准确。模型把 KV Cache 说成“缓存 previous inference tasks”或“缓存训练数据/常用信息”，这偏离了 LLM 推理里的真实含义。

模式 质量观察
FP16 表达更稳定，但把 KV Cache 泛化成“缓存以前推理任务结果”，概念不准确。
INT4 没有明显崩坏，但同样出现概念偏差，并且把 cache 与训练数据访问混在一起。

因此，本次质量结论应写成：INT4 未出现明显灾难性退化，但在事实准确性上 FP16 和 INT4 都受 1.7B 小模型本身能力限制。

9. 实践注意事项

  • 首次运行如果包含模型下载，加载时间会被网络和缓存污染；要复跑一次再记录加载速度。
  • 显存建议同时记录 memory_allocated、memory_reserved 和 max_memory_allocated。
  • 比较生成速度时，输入 prompt、max_new_tokens、采样参数要保持一致。
  • 回答质量不能只看是否能输出，还要检查事实准确性、完整性、是否有幻觉或概念混淆。
  • INT4 更适合显存紧张、模型更大、部署密度更高的场景；小模型单卡测速时不一定更快。

10. 面试表达

可以这样说：

我做过 FP16 和 INT4 的量化推理对比。以 SmolLM2-1.7B-Instruct 为例，INT4 把加载后显存从约 3.19GB 降到 1.04GB，峰值显存从约 3.27GB 降到 1.11GB，显存节省约三分之二。但在这台机器上，INT4 的生成速度从 53.61 tokens/s 降到 42.32 tokens/s，加载也从 2.85s 增加到 4.52s。我的理解是，INT4 主要降低权重显存和访存压力，但 bitsandbytes 4bit 推理存在反量化和 kernel 开销，小模型上不一定能换来速度提升。质量方面没有明显格式崩坏，但准确性仍取决于模型本身能力和量化误差，需要结合任务人工评估。
