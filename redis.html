42. Redis 为什么快？

Redis 快的主要原因包括：数据在内存中、核心命令执行路径高效、采用事件驱动 IO 模型、数据结构设计紧凑、避免频繁磁盘随机 IO。

Redis 单线程主要指命令执行的主路径单线程，这减少了锁竞争；现代 Redis 在网络 IO、持久化等方面也有多线程辅助能力。

43. Redis 持久化 RDB 和 AOF 区别？

对比项RDBAOF
方式周期性生成快照记录写命令日志
恢复速度通常较快取决于日志大小
数据丢失窗口可能丢失最近一次快照后的数据通常更小
适合场景备份、灾备、快速恢复更重视数据安全

生产环境常见做法是 RDB 和 AOF 结合使用，兼顾恢复速度与数据安全。

44. 缓存穿透、击穿、雪崩是什么？

问题含义常见方案
穿透查询不存在的数据，缓存和数据库都没有缓存空值、布隆过滤器、参数校验
击穿热点 key 过期，大量请求打到数据库互斥锁、逻辑过期、热点 key 不过期
雪崩大量 key 同时失效或 Redis 故障过期时间随机化、多级缓存、限流降级

45. Redis 分布式锁怎么实现？

基本做法是使用 SET key value NX PX milliseconds 原子加锁，释放锁时用 Lua 脚本判断 value 是否匹配，避免误删其他客户端的锁。

SET lock:order:1 request-id NX PX 30000

注意锁超时、业务执行时间、续期、可重入、主从切换一致性等问题。对强一致要求极高的场景，应谨慎评估 Redis 锁是否足够。

46. Redis 主从、哨兵、集群区别？

模式作用
主从复制读写分离、数据副本、提升读能力
哨兵 Sentinel监控主从、故障转移、自动选主
Cluster数据分片、横向扩展、高可用

Redis Cluster 通过 hash slot 分片，不同 key 会被映射到不同槽位，再由不同节点负责。

66. Redis maxmemory 有什么作用？

maxmemory 用于限制 Redis 可使用的最大内存。达到限制后，Redis 会根据配置的淘汰策略删除 key，或者在 noeviction 策略下拒绝新的写入。

容器化部署时，maxmemory 应小于容器内存上限，给系统、复制缓冲、客户端缓冲等留出余量。

67. Redis 内存淘汰策略有哪些？

常见策略包括 allkeys-lru、volatile-lru、allkeys-lfu、volatile-lfu、allkeys-random、volatile-random、volatile-ttl、noeviction。

allkeys 面向所有 key，volatile 只面向设置了过期时间的 key。缓存场景常用 LRU 或 LFU，不能丢数据的场景要谨慎使用淘汰。

68. Redis 内存异常增长如何排查？

  1. 查看 INFO memory，确认 used_memory、碎片率、客户端缓冲等指标。
  2. 用 SCAN 或分析工具找大 key、热 key、无过期 key。
  3. 检查是否有列表、集合、Stream 持续追加未清理。
  4. 核对过期策略、业务缓存 key 设计和数据写入量。

69. Redis 如何实现滑动窗口限流？

可以用 Sorted Set 记录请求时间戳，每次请求删除窗口外数据，统计窗口内请求数，未超过阈值则写入当前时间戳并放行。

生产中通常用 Lua 脚本把清理、计数、写入、设置过期时间做成原子操作，避免并发误判。

235. 为什么用 Redis 而不是本地缓存？

本地缓存（Caffeine/Guava）访问快，但多实例间不共享，更新不一致，重启丢失。

Redis 提供跨进程、跨节点的共享缓存，适合分布式系统；常配合本地缓存做多级缓存。

236. Redis 除了缓存还能做什么？

  • 分布式锁、限流计数
  • 排行榜（ZSet）、延时任务（ZSet 或 Stream）
  • 轻量消息队列（List/Stream）
  • 会话存储、Bitmap 签到、HyperLogLog UV 统计

237. Redis Module 是什么？

Redis Module 允许通过动态库扩展 Redis 能力，例如 RediSearch、RedisJSON、RedisGraph 等，在不改核心代码的情况下增加新数据类型和命令。

238. Redis 可以做消息队列吗？

可以，但不适合替代 RocketMQ/Kafka 等专业 MQ。List 做简单队列，Stream 支持消费组和 ACK，Pub/Sub 适合广播但不持久化。

轻量异步、允许丢消息的场景可用 Redis；强可靠、堆积量大应选专业 MQ。

239. Redis 如何实现延时任务？

常见方案：ZSet 以执行时间戳为 score 存储任务 ID，定时扫描 ZRANGEBYSCORE 到期任务；或使用 Redisson DelayedQueue、Stream 延时消费。

240. Redis String 适合哪些场景？

  • 缓存对象 JSON、计数器（INCR）
  • 分布式锁、限流令牌
  • Session、验证码、短链映射

241. Redis Hash 存对象什么时候更合适？

对象字段较多且需要部分更新时，Hash 比整段 JSON String 更省带宽、支持 HGET/HSET 单字段操作。

字段少、整体读写为主时，String 存 JSON 更简单。

242. 购物车信息用 String 还是 Hash 存储更好？

推荐 Hash：cart:{userId} 下 field 为 skuId，value 为数量，支持单商品增减而不重写整个购物车。

243. Redis 如何实现排行榜？

使用 ZSet，member 为用户/商品 ID，score 为分数。支持 ZREVRANGE 取 TopN、ZRANK 查名次、ZINCRBY 加分。

244. Redis 有序集合为什么使用跳表？

跳表在平均 O(log N) 时间内支持范围查询和排名，实现比红黑树简单，且与 Redis 的 ziplist 等紧凑编码配合良好。

245. Bitmap 适合什么场景？

大量布尔状态：用户签到、日活统计、布隆过滤器底层、权限位图。极省内存，支持 BITCOUNT、BITOP。

246. HyperLogLog 适合什么场景？

海量 UV/独立访客统计，用固定约 12KB 内存估算基数，允许约 0.81% 误差，不适合需要精确计数的场景。

247. GEO 适合什么场景？

附近的人/店、距离计算、地理围栏。底层基于 ZSet 存储经纬度编码。

248. Redis 事务为什么不支持回滚？

Redis 追求高性能和简单语义，命令错误在入队时不会阻止执行；运行时错误不会回滚已执行的命令。复杂原子性应使用 Lua 脚本。

355. Redis String 的 SDS 是什么？

SDS（Simple Dynamic String）是 Redis 自定义字符串结构，记录 len、free，支持 O(1) 取长度、二进制安全、预分配减少扩容。

356. Redis Hash 适合什么场景？

对象属性存储、购物车、用户信息字段级更新。小 Hash 用 ziplist 紧凑存储，大 Hash 转 hashtable。

357. Redis ZSet 底层结构是什么？

元素少时用 ziplist；元素多时使用跳表（按 score 排序）+ 哈希表（member → score）双结构，兼顾范围查询与 O(1) 单点查找。

358. Redis 跳表是什么？

多层有序链表，通过随机层高实现近似平衡，支持 O(log N) 插入、删除、范围查询，实现比平衡树更简单。

359. Redis Big Key 和 Hot Key 分别是什么？

Big Key：value 过大或集合元素过多，导致阻塞、迁移慢。Hot Key：访问频率极高，造成单分片/单节点压力过大。

360. 如何发现和治理 Big Key？

  • 发现：redis-cli --bigkeys、RDB 分析、内存采样
  • 治理：拆分 key、压缩、异步删除（UNLINK）、限制集合大小

361. 如何发现和治理 Hot Key？

  • 发现：monitor 采样、hotkeys 命令、客户端上报
  • 治理：本地缓存、key 副本扩散、读写分离、限流

362. Redis Pipeline 是什么？

客户端批量发送多条命令再批量读取结果，减少 RTT，但不保证事务原子性。适合大量独立读写。

363. Redis Lua 脚本为什么能保证原子性？

Redis 单线程执行模型下，Lua 脚本执行期间不会插入其他命令，因此脚本内多条命令作为一个整体连续执行。

常用于分布式锁释放、限流、库存扣减。脚本应短小，避免阻塞主线程。

364. Redis Stream 是什么？适合什么场景？

Stream 是 Redis 5.0 引入的持久化消息结构，支持消费组、ACK、消息 ID 有序，适合轻量 MQ、日志流、事件溯源。

365. Redis Pub/Sub 和 Stream 有什么区别？

对比项Pub/SubStream
持久化否，订阅者离线丢消息是
消费组无支持
适用实时广播通知可靠队列、事件流

366. Redis 主从复制全量同步和增量同步是什么？

全量同步：从库首次连接或 offset 差距过大时，主库 RDB 快照 + 缓冲期间命令。增量同步：基于 replication backlog 传播 offset 之后的写命令。

367. Redis Cluster hash slot 是什么？

Cluster 将 key 空间分为 16384 个槽位，每个 master 负责一部分 slot。key 通过 CRC16 对 slot 取模路由，支持 slot 迁移实现扩缩容。

368. Redis 脑裂问题是什么？如何降低风险？

网络分区导致出现多个 master 同时写入，恢复后可能丢数据。治理：min-replicas-to-write、合理超时、Sentinel/Cluster 多数派选主。
