React 是什么？常见组件有哪些？

React 是 Facebook 开源的用于构建用户界面的 JavaScript 库，核心思想是组件化 + 声明式 UI + 单向数据流，通过 Virtual DOM 和 Diff 算法高效更新页面。

核心特点

  • 组件化：UI 拆成独立、可复用的组件，每个组件管理自己的状态和视图。
  • 声明式：描述「UI 应该是什么样」，React 负责更新 DOM，而非手动操作节点。
  • Virtual DOM：内存中维护虚拟树，状态变化时 Diff 后最小化真实 DOM 更新。
  • 单向数据流：数据从父组件经 props 流向子组件，状态变更可预测。
// 函数组件 + Hooks（现代写法）
function Welcome({ name }) {
  const [count, setCount] = useState(0);
  return (
    <div>
      <h1>Hello, {name}</h1>
      <button onClick={() => setCount(count + 1)}>{count}</button>
    </div>
  );
}

常见组件类型

类型 说明 示例
函数组件用函数定义，配合 Hooks 管理状态与副作用function App() { ... }
类组件旧写法，class + render()，现多用于维护老项目class App extends Component
展示组件只接收 props 渲染 UI，无业务状态Button、Card、Table
容器组件负责数据获取、状态管理，传给展示组件UserListContainer
高阶组件 HOC函数接收组件返回增强组件，复用逻辑withAuth(Component)
复合组件多个子组件组合成完整 UI 模式Modal + Modal.Header + Modal.Body

常用内置 / 生态组件与 Hooks

名称 作用
useState组件内部状态
useEffect副作用：请求、订阅、DOM 操作
useContext跨层级共享数据，避免 props 层层传递
React Router单页应用路由：Route、Link、Navigate
Form 组件受控组件 input value + onChange，或 React Hook Form
List 组件map 渲染列表，key 必须唯一稳定

面试回答：React 是构建 UI 的组件化库，声明式渲染 + Virtual DOM + 单向数据流。现代开发以函数组件 + Hooks 为主；常见组件分展示/容器组件，配合 Router、Form、List 等模式；状态用 useState/useEffect，跨层共享用 Context 或 Redux/Zustand。
