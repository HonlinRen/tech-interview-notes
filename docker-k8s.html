495. Docker 是什么？解决了什么问题？

Docker 是一种容器化技术，可以把应用程序、运行环境、依赖库一起打包成镜像，然后在不同机器上以容器方式运行。

它主要解决环境不一致、部署复杂、服务隔离和快速扩缩容等问题。

面试回答：Docker 让应用和运行环境一起交付，避免「我本地可以运行，线上不行」的问题。

496. 镜像 Image 和容器 Container 有什么区别？

镜像：静态模板，类似安装包。
容器：镜像运行后的实例，类似正在运行的进程。

镜像是静态的，容器是运行中的实例。一个镜像可以启动多个容器。

497. Dockerfile 是什么？常见指令有哪些？

FROM openjdk:17
WORKDIR /app
COPY target/demo.jar app.jar
EXPOSE 8080
CMD ["java", "-jar", "app.jar"]
  • FROM：指定基础镜像
  • WORKDIR：工作目录
  • COPY：复制文件
  • EXPOSE：声明端口
  • CMD：启动命令

498. COPY 和 ADD 有什么区别？

COPY 单纯复制文件。ADD 还支持自动解压 tar 和 URL 下载。日常推荐优先使用 COPY，语义更清晰。

499. CMD 和 ENTRYPOINT 有什么区别？

CMD 是默认启动命令，可被 docker run 参数覆盖。ENTRYPOINT 是固定入口，适合启动方式固定的镜像。

500. Docker 容器之间如何通信？

  • 宿主机端口映射：docker run -p 8080:8080
  • Docker Network 自定义网络
  • Compose 中通过 service name 访问，如 redis:6379

501. Docker 数据持久化怎么做？

使用 Volume 或 bind mount 挂载宿主机目录，例如：

docker run -v /host/data:/container/data mysql

有状态服务（MySQL、Redis）必须持久化，不能只依赖容器内文件系统。

502. Docker Compose 是什么？

用于编排多个容器的工具，通过 docker-compose.yml 定义服务、网络、端口、依赖，一键启动整套环境，适合本地开发和测试。

503. Docker 容器如何查看日志和排查问题？

docker ps
docker logs container_id
docker exec -it container_id /bin/bash
docker inspect container_id
docker stats

先看容器是否运行，再看日志，必要时进入容器检查环境变量、网络和进程。

504. Docker 镜像怎么优化？

  • 使用更小基础镜像（alpine、distroless）
  • 多阶段构建，只保留运行产物
  • .dockerignore 排除无关文件
  • 合并 RUN 层、清理缓存

505. Kubernetes 是什么？解决了什么问题？

K8s 是容器编排平台，管理大规模容器的部署、扩缩容、服务发现、故障自愈、滚动发布和配置管理。

Docker 解决单机容器运行，K8s 解决多机器、多服务的编排治理。

506. Pod 是什么？

Pod 是 K8s 最小调度单位，通常一个 Pod 运行一个业务容器。Pod 内容器共享网络和存储卷。

507. Pod 和 Container 有什么区别？

Container 是运行应用的进程隔离环境；Pod 是 K8s 对容器的封装和调度单位，K8s 调度 Pod 而非直接调度 Container。

508. Deployment 是什么？

Deployment 管理无状态应用的 Pod 副本数、滚动更新和回滚，是微服务最常用的部署对象。

spec:
  replicas: 3
  template:
    spec:
      containers:
        - name: demo-app
          image: demo-app:1.0

509. Service 是什么？为什么需要 Service？

Pod IP 不稳定，Service 为一组 Pod 提供稳定访问入口和负载均衡，通过 selector 关联 Pod，集群内 DNS 访问。

510. ConfigMap 和 Secret 有什么区别？

ConfigMap 存放普通配置；Secret 存放密码、Token、证书等敏感信息。两者都支持环境变量或 Volume 挂载注入容器。

511. K8s 如何做滚动发布和回滚？

kubectl rollout status deployment/demo-app
kubectl rollout undo deployment/demo-app

Deployment 逐步创建新 Pod、下线旧 Pod，异常时可快速回滚到上一版本。

512. K8s 的 HPA 和 VPA 有什么区别？

类型方向场景
HPA水平：增减 Pod 数量流量波动、无状态服务
VPA垂直：调整单 Pod 资源资源申请不合理

513. request 和 limit 有什么区别？

request 是调度保证资源，limit 是容器可用上限。CPU 超限被限流，内存超限可能 OOMKilled。

514. Pod 一直启动失败，你会怎么排查？

kubectl get pods
kubectl describe pod pod-name
kubectl logs pod-name
kubectl logs pod-name --previous

常见原因：ImagePullBackOff、CrashLoopBackOff、配置缺失、健康检查失败、资源不足、应用自身异常。
